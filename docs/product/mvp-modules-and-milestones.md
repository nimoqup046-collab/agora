# AGORA MVP 模块与里程碑（执行版）

## Sprint 0（本周）

目标：把“讨论”变成“可执行工程面”。

- 输出可行性与市场评估
- 输出 MVP 技术蓝图
- 定义 API 最小合同与状态机
- 补齐最小可运行骨架（web/api）

验收：

- 仓库内有可直接跟随的实施文档
- API 服务可启动且健康检查通过

## Sprint 1（第 1-2 周）

目标：会话与任务主链路打通。

- Workspace / Session / Task CRUD
- 任务状态机与审批流
- 代理回合记录（AgentTurn）
- 前端基础面板（会话列表、任务详情）

验收：

- 能从界面创建任务并看到状态变化
- 高风险任务可被人工审批继续

## Sprint 2（第 3-4 周）

目标：接入真实代理与 PR 闭环。

- 接入至少 2 个模型提供方（通过 adapter）
- 产出代码补丁证据与测试结果
- GitHub Draft PR 创建 + 评论回写
- 审查结论回写任务日志

验收：

- 至少完成 5 个真实任务并产出 PR
- 每个任务都有可追溯证据链

## Sprint 3（第 5-6 周）

目标：稳定性、成本、治理。

- 成本路由策略与预算阈值
- 幂等重试与失败恢复
- 基础仪表盘（成功率/返工率/成本）
- 关键风险演练（权限、误操作、回滚）

验收：

- 20 个样本任务评估报告
- Go / No-Go 结论与下一阶段计划

## 模块责任映射

- `apps/api`: 业务编排与 API 合同
- `packages/adapter`: 多模型统一适配
- `packages/orchestrator`: 任务调度与状态机
- `packages/memory`: 会话记忆与检索
- `apps/web`: 协作终端与可观测视图
